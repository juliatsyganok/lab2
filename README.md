# Лабораторная работа: Дескрипторы в Python

## Описание

В данной работе реализован класс `Task`, представляющий задачу с управляемыми атрибутами. Для контроля корректности данных при записи в атрибуты использованы дескрипторы протокола Python.

---

## Реализованные дескрипторы

### `empty`

Базовый дескриптор для строковых полей. При присвоении значения выполняет две проверки:

- значение является экземпляром типа `str`, иначе выбрасывается `TypeError`;
- строка не пустая и не состоит исключительно из пробельных символов (проверка через `.strip()`), иначе выбрасывается `ValueError`.

Используется для поля `description` класса `Task`.

---

### `reading`

Дескриптор, наследующий `empty`. Расширяет родительский `__set__` ограничением на однократную запись: после первоначального присвоения повторная запись в атрибут невозможна — выбрасывается `AttributeError`. Все строковые проверки родительского класса сохраняются через вызов `super().__set__()`.

Используется для поля `id` класса `Task`, идентификатор которого не должен изменяться после создания объекта.

---

### `choice`

Дескриптор для ограничения значения допустимым набором. Принимает список разрешённых значений в качестве аргумента при инициализации. При попытке записи значения, отсутствующего в списке, выбрасывается `ValueError`.

Используется для полей:
- `priority` — допустимые значения: `"low"`, `"medium"`, `"high"`;
- `status` — допустимые значения: `"new"`, `"in_progress"`, `"done"`.

---

## Использование `property` в классе `Task`

Помимо дескрипторов, в классе `Task` применён встроенный механизм `property`:

- `created_at` — возвращает дату и время создания объекта. Сеттер не определён, атрибут доступен только для чтения;
- `is_ready` — вычисляемое свойство, возвращает `True` если задача не завершена (`status != "done"`) и не имеет низкого приоритета (`priority != "low"`). Значение не хранится, вычисляется при каждом обращении.

---

## Структура атрибутов класса `Task`

| Атрибут | Дескриптор | Описание |
|---|---|---|
| `id` | `reading` | Строковый идентификатор, записывается однократно |
| `description` | `empty` | Непустая строка описания задачи |
| `priority` | `choice` | Приоритет из фиксированного набора значений |
| `status` | `choice` | Статус задачи из фиксированного набора значений |
| `created_at` | `property` | Дата и время создания, только для чтения |
| `is_ready` | `property` | Вычисляемый признак готовности к выполнению |
